# Workflow de Développement

## Outils Quotidiens

- **Claude Code** : Outil principal (usage intensif quotidien)
- **Cursor** : IDE
- **VS Code** : IDE alternatif
- **Hostinger** : Hébergement

## Processus de Développement

### Étapes

1. **PRD (Product Requirements Document)**
   - Créé avec agent IA
   - Définit vision produit et specs

2. **Génération Base Technique**
   - Agent génère structure front-end
   - Agent génère structure back-end
   - Architecture de base complète

3. **Review Humaine**
   - Hugo valide l'architecture
   - Corrections si nécessaire

4. **Logique Métier**
   - Implémentation manuelle par Hugo
   - Code business critique

5. **Amélioration Continue**
   - Incrémentation autonomie agent
   - Optimisation qualité
   - Réduction agents IA utilisés

## Objectif du Workflow

**Maximiser qualité et autonomie, minimiser nombre d'agents IA nécessaires.**

Le workflow évolue constamment pour devenir plus autonome et qualitatif.
